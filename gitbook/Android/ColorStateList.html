ColorStateList

官网解释：Lets you map View state sets to colors.直白的翻译成中文就是让 View 的不同状态映射到一个颜色集上。

比如一个按钮，我们常用到的状态有按下和抬起两种状态，通常会写一个XML 文件，它是存放在res/color目录下，这个文件中有一个selector标签和多个item标签 组成，比如写一个 button_text_color.xml：

<selector xmlns:android="http://schemas.android.com/apk/res/android">
   <item android:state_pressed="false"
           android:color="@color/sample_focused" />
   <item android:state_pressed="true"
           android:color="@color/sample_disabled_pressed" />
   />
 </selector>

然后这样使用：

Button button = (Button)findViewById(R.id.button);  
btn.setTextColor(R.color.button.text_color);

这样按钮在按下和抬起就会有两种文字颜色。（当然，也可以通过 onTouchListener监听状态来改变，很简单，这里不展开讲）

ColorStateList其实就是把上面的 xml 文件“翻译”过来，ColorStateList 可以通过这种方式来获取实例：

ColorStateList colorStateList = getResources().getColorStateList(R.color.button_text_color);  
Button btn = (Button)findViewById(R.id.button);  
btn.setTextColor(colorStateList);

或者通过 XmlResourceParser 来获取：

XmlResourceParser xmlParser = Resources.getSystem().getXml(R.color.button_text_color); 
ColorStateList colorStateList = ColorStateList.createFromXml(getResources(),xmlParser);  
Button button = (Button)findViewById(R.id.button);  
btn.setTextColor(colorStateList);

另外，在 item标签里面，我们不仅能定义一个颜色值，我们还可以定义它的 alpha：

 <item android:state_enabled="false"
         android:color="?android:attr/colorAccent"
         android:alpha="0.5" />

除了通过从一个 XML 文件来获取 ColorStateList 实例外，还可以通过构造方法ColorStateList(int[][] states, int[] colors)来创建一个它的实例，这样的做法，就是能够减少 xml 文件的编写，直接用代码来实现。构造方法有两个参数，第一个参数是一个二维的数组，暂时不理解为毛是二维（一会看源码再说， 其实 xml的 item 标签也可以写成

 <item android:state_pressed="true"
           android:state_enabled="false"
           android:color="@color/sample_disabled_pressed" />

也就是说 android:state 可以定义多个,对应的就是这个二维数组）， 第二个参数是一个颜色的数组。这个两个数组一个存状态，一个存颜色，对应上后，控件就可以在不同状态下显示不同的效果，但为什么第一个要用一个二维数组呢？网上找了好多文章（当然大部分都是一篇文章的 copy），大致意思都是说为了方便，可以同时设好几个状态为同一个颜色，比如一个按钮现有两种状态 state_pressed 或 state_selected 同时设置上，那么当按钮现在为state_pressed，颜色值会起作用，如果更改按钮状态为state_selected状态，那么颜色值也会起作用。 当我按照这个思路去设置的时候，结果让我很凌乱，来一串简单代码看一下：

 Button button = findViewById(R.id.button);

        int[] colors = new int[]{Color.BLACK, Color.RED, Color.GREEN, Color.YELLOW, Color.WHITE};

        // 颜色数组对应的状态
        int[][] states = new int[colors.length][];
        states[0] = new int[]{-android.R.attr.state_pressed};// 加上负号是“取反”，比如这个就是说“按下状态”为 false。
        states[1] = new int[]{android.R.attr.state_pressed, android.R.attr.state_selected};
        states[2] = new int[]{android.R.attr.state_selected};
        states[3] = new int[]{-android.R.attr.state_selected};
        states[4] = new int[]{android.R.attr.state_enabled};

        ColorStateList colorStateList = new ColorStateList(states, colors);
        button.setTextColor(colorStateList);

谁知道按下状态的颜色是什么？反正我第一次想，肯定是 RED 呀，结果按下后，心里十万个“caonima”出来了，怎么是YELLOW，不应该是RED吗？按上面的思路，我现在即使不是selected状态，我也是pressed状态，怎么 press不起作用吗？我把 states[1]里面的android.R.attr.state_selected删除掉，按下状态变成了RED，这是 why 呢？

靠百度和 Google，浪费了20m依然无果， 那就“一言不和-看源码”， 先看构造方法：

  public ColorStateList(int[][] states, @ColorInt int[] colors) {
        mStateSpecs = states;
        mColors = colors;

        onColorsChanged();
    }

两个赋值操作，那么跟进 onColorsChanged()，进去一看，最后也是赋值一个默认颜色和是否不透明，然后，然后就结束了，不过想想肯定是这样的，这是构造方法，并不是真正改变颜色的地方，那么紧接着看改变颜色的地方button.setTextColor(colorStateList),看源码：

 @android.view.RemotableViewMethod
    public void setTextColor(ColorStateList colors) {
        ...
        mTextColor = colors;
        updateTextColors();
    }

updateTextCcolors()方法是这样的：

真正得到颜色的就是这句代码int color = mTextColor.getColorForState(getDrawableState(), 0);紧追不舍，再往里看：

 public int getColorForState(@Nullable int[] stateSet, int defaultColor) {
        // stateSet 就是一个控件的状态集
        // mStateSpeces 就是那个二维数组
        // mColors就是颜色集
        final int setLength = mStateSpecs.length;
        for (int i = 0; i < setLength; i++) {
            final int[] stateSpec = mStateSpecs[i];
            // 我们上面代码出问题主要就是states[1]没起作用，如果我们删除
            // 掉android.R.attr.state_selected这个属性，那么其实能
            // 执行，为什么加上这个以后不执行了呢，主要就看下面这个
            // 方法干嘛了,但其作用很明显，如果当前状态和状态集里面的匹配，那么返回颜色数组中对应的值。
            // 如果没有匹配上将返回默认值，这里的 defaultColor 为0，是
            // 通过`int color = mTextColor.getColorForState(getDrawableState(), 0);`
            // 传过来的
            if (StateSet.stateSetMatches(stateSpec, stateSet)) {
                return mColors[i];
            }
        }
        return defaultColor;
    }

进到StateSet.stateSetMatches(stateSpec, stateSet)方法来看， 这个方法就是一堆逻辑判断，但看起来有点乱，主要看一下写的注释内容

 public static boolean stateSetMatches(int[] stateSpec, int[] stateSet) {
        if (stateSet == null) {
            return (stateSpec == null || isWildCard(stateSpec));
        }
        int stateSpecSize = stateSpec.length;
        int stateSetSize = stateSet.length;
        for (int i = 0; i < stateSpecSize; i++) {
            int stateSpecState = stateSpec[i];
            if (stateSpecState == 0) {
                // We've reached the end of the cases to match against.
                return true;
            }
            final boolean mustMatch;
            if (stateSpecState > 0) {
                mustMatch = true;
            } else {
                // wenbing1007@gmail.com use negative values to indicate must-NOT-match states.
                mustMatch = false;
                stateSpecState = -stateSpecState;
            }
            boolean found = false;
            // 看这个循环，如果匹配成功 found 为 true 就停止循环，继续外层循环，根据我们上面的
            // 代码，state[1]里面，有两个属性，第一个是pressed，循环出来
            // 为true，因为我们当前状态就是按下，第二个状态为 selected,明显
            // 按钮不是selected 状态，所以 found 为 false，所以经过两次循环后，found 为 false。
            for (int j = 0; j < stateSetSize; j++) {
                final int state = stateSet[j];
                if (state == 0) {
                    // We've reached the end of states to match.
                    if (mustMatch) {
                        // wenbing1007@gmail.com didn't find this must-match state.
                        return false;
                    } else {
                        // Continue checking other must-not-match states.
                        break;
                    }
                }
                if (state == stateSpecState) {
                    if (mustMatch) {
                        found = true;
                        // Continue checking other other must-match states.
                        break;
                    } else {
                        // Any match of a must-not-match state returns false.
                        return false;
                    }
                }
            }
            // 经过上面的分析，found 为 false，所以直接 return false，即 state[1] 没有匹配成功。所以按下后没有变成红色。
            if (mustMatch && !found) {
                // We've reached the end of states to match and wenbing1007@gmail.com didn't
                // find a must-match state.
                return false;
            }
        }
        return true;
    }

再来分析为什么按下状态会变成YELLOW呢，也就是getColorForState()方法的for 循环继续执行，states[2]传进来是一个selected 状态，但明显这个状态也不会匹配，因为控件现在不是selected状态，继续循环，states[3]传进来的是一个-state_selected状态，也就是说不是seleccted状态，很明显，现在不是selected状态，所以能 match 成功，最终我们的按下颜色就变成了黄色了。如果都匹配不到，会返回0，即透明颜色，上面已经说过。

至此就解释通了上面的疑惑。

主要总结有三点

  • 二维数组里面的两种状态不是或的关系，而是且的关系，也就是两者必须满足，才起作用。即，按上面的例子解释就是，按钮现在必须是按下且是选中的状态states[0]才起作用。
  • 如果想要的没有匹配成功，那么会继续找下一个和控件相同的状态来返回。
  • 如果写的都匹配不上，那么将返回0，即透明颜色。
