gradle plugin编写

插件编写可以在三个地方定义插件

1.在 build.gradle 中定义

class ScriptPlugin implements Plugin<Project>{
    @Override
    void apply(Project target) {
        //添加一个任务
        target.task('script0', group: 'script') {
            doLast {
                println '>>> script plugin doLast call'
            }
        }
    }
}

apply plugin:ScriptPlugin

通过执行./gradlew script0

> Task :app:script0
>>> script plugin doLast call

2.在项目的buildSrc目录，这里知道就行，一般不在这里面写。

3.在独立的项目上编写，并上传到maven仓库，通过依赖的方式使用。这种方式是最常见，也是用得最多，最方便的一种方式。

这里使用IntelliIdea ide 来创建。

点击Next

这里随便起一个名字，点击finish后，等待项目创建

创建完成后，在groovy目录下，新建一个包

在build.gradle文件中把gradle sdk依赖进来，修改dependences如下，这里为了兼容性，我们使用本地groovy，。

dependencies {
//compile 'org.codehaus.groovy:groovy-all:2.3.11'

compile localGroovy()

    testCompile group: 'junit', name: 'junit', version: '4.12'
    implementation gradleApi()
}

新建一个groovy class文件，这里起名为plugin0，让它实现Plugin接口，这里泛型定义Project，并实现它的方法，在方法里面简单打印一句话plugin0 apply method called.,这样这个插件就实现了，那要怎么使用呢？

package com.test.gradle.plugin

import org.gradle.api.Plugin
import org.gradle.api.Project

class Plugin0 implements Plugin<Project>{
    @Override
    void apply(Project project) {
        print "plugin0 apply method called."
    }
}

首先在resource目录下，新建META-INF目录，再建立gradle-plugins子目录，然后根据包名及插件名创建properties文件，并在文件中写入implementation-class=com.test.gradle.plugin.Plugin0

完成这步以后，就可以上传插件，这里先上传到本地仓库，简单配置如下：

plugins {
    id 'groovy'
    id 'maven-publish'
}

publishing {
    publications {
        Production(MavenPublication) {
            from components.java
        }
    }
}

group 'org.test.plugins'
version '1.0'

剩下的就是使用插件。

新建一个Android项目，在project的build.gradle中添加插件的引用

// Top-level build file where you can add configuration options common to all sub-projects/modules.
buildscript {
    repositories {
        maven { url '/Users/wenbing/.m2/repository/' }
        google()
        jcenter()

    }
    dependencies {
        classpath "com.android.tools.build:gradle:4.1.1"
        classpath 'org.test.plugins:untitled2:1.0'

        // NOTE: Do not place your application dependencies here; they belong
        // in the individual module build.gradle files
    }
}

allprojects {
    repositories {
        maven { url '/Users/wenbing/.m2/repository' }
        google()
        jcenter()
    }
}

task clean(type: Delete) {
    delete rootProject.buildDir
}

在module的build.gradle文件中添加我们定义好的插件id，``

plugins {
    id 'com.android.application'
    id 'com.test.gradle.plugin.Plugin0'
}
