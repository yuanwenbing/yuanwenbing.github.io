Hexo+Github搭建个人博客（Mac系统）

前言

先简单介绍一下这两个东西是什么玩意：

Hexo

直接看官网解释，A fast, simple & powerful blog framework，说的很明白，就是一个搭建博客的框架。当然 hexo 的使用需要基于 node.js，所以我们需要首先安装 node，可以通过官网来下载，安装上去就好，Mac 版本下载下来 pkg 文件直接安装就好。另外需要需要版本管理工具 git，这个macOs 默认是安装好的。

Github

这个就不说了，是程序员就应该知道，世界上最知名有项目托管平台。

开始

创建仓库

首先要注册一个Github 账号，新建一个仓库名为xxx.github.io 的仓库，xxx 是你的用户名，比如你的用户名是 hello，那么以后你的博客地址就是http://hello.github.io 。这里注意一点，申请的仓库也许不会立即生效，需要0~30分钟左右，耐心等待就好。

配置 ssh key

  • 在 Termial 中执行 ssh-keygen -t rsa命令生成一个 key。
$ ~/.ssh ssh-keygen -t rsa

$ Enter file in which to save the key (/Users/yuan/.ssh/id_rsa):
$ Enter passphrase (empty for no passphrase):
$ Enter same passphrase again:
Your identification has been saved in /Users/yuan/.ssh/id_rsa.
Your public key has been saved in /Users/yuan/.ssh/id_rsa.pub.
$ The key fingerprint is:
SHA256:4Jtqrxrjy/FjO7MtFFPZDr2Xw77dsy4K66JMsN/Kw48 yuan@yuanwenbingdeMacBook-Pro.local
The key's randomart image is:
+---[RSA 2048]----+
|       +         |
|      + o        |
|     ..o o .     |
|    o. .o =      |
|  .  o. So .     |
|   o.  o  .      |
|  =o. o .  o .   |
| o XX=.  o. o o  |
|  ==E#=oo .. ooo |
+----[SHA256]-----+

一直按回车就好，当然你也可以根据自己需要更改一些东西，比如添加 passphrase，这个会在你发布的时候，让你输入这个密码。这样的话，别人就不能用你的电话直接发布了。

  • 生成完以后会在~/.ssh目录里有一个id_rsa.pub 的文件。
$ cat id_rsa.pub
$ ssh-rsa AAAAB3NzaC1yc2EAAAADAQABAAABAQC8mBmbwEzoDB7rFF+ETMhFbbDxxAuohXRxo0IaqDfdIdVSHufQ5qRUIi+INpVqNvDapYKZ0YBaEnC1w9RegXFFUVz5g9n3srm7AtBBxUVd3roUrNclaSfNy+ddH4ESqDg7T3+f2Vaf27JL29Rb+vKWHPAzB7HL39lmYCIqR+jdGxq1EwP2WIB/HesxwD4t/WTfCTYVDCEowIpvhgaCHeQGke1jZ26ok59aREXMYlduAqbNLHgXTatOLHBqLKZccYezPYH183XPi7Aw6MxV2UFodAA60xdZSlg2nY8NhhoWSKIXZ4nE/etnwSTpVV+RgpGx1keuXbFl9Pgwcge3sz xxx-Pro.local

通过 cat 命令获取到这个文件的内容，复制它。打开github.com 点击右上角的头像 --> Settings --> SSH and GPG keys --> New SSH key，Title 可以随便写，key 写我们复制过来的key。

  • 用以下命令测试一下是否通过
$ ssh -T git@github.com

如果结果是

$ git@github.com: Permission denied (publickey).

说明没有配置成功，按上面方式再重新搞一下试试。如果出现

$ Hi xxx! You've successfully authenticated， but GitHub does not provide shell access.

恭喜，你成功了。

配置 hexo

  • 使用npm命令安装：npm install -g hexo
  • 安装完成后，在电脑上建立一个文件夹，这个文件夹就是以后本地存放博客和其它配置文件的地方，比如在 work 目录下建立一个叫 hexo 的目录。
$ cd /work/hexo
$ hexo init

此时hexo 会执行一些初始化的工作，会下载一下必须的组件，可能会需要一点时间。

  • 执行 hexo g 命令，相当于build
  • 执行 hexo s 会启动一个本地服务
~/Work/hexo hexo s
INFO  Start processing
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

在浏览器输入这个 http://localhost:4000 地址，会直接预览本地的博客。如果启动失败，由可能是端口地址被占用了。

上传到 Github

首先要修改一下配置文件，它就在我们新建的目录里面_config.yml，修改一下 deploy为

deploy:
  type: git
  repository: git@github.com:xxx/xxx.github.io.git //注意此处只修改 xxx 为你的用户名即可。
  branch: master

注意：.yml的文件 key冒号后面一定要有空格。

这时候我们执行hexo d 会发现报错：

Deployer not found: github 或者 Deployer not found: git

这时候需要安装一个插件，同样用npm来安装

npm install hexo-deployer-git --save

安装完成后再次执行hexo d，就发布成功了
