activity-alias

有时候做 app 的时会有这样的要求，要求提供一种快捷方式，能够快速进入到 app 的某个界面，比如有一些 app 有一个主程序的入口，有一个“扫一扫”的入口，两个都放在 launcher 中，“扫一扫”入口不仅可以通过快捷方式进入，也可以从程序中的某一个入口进去，或者说也是对程序的一种重复使用。 这样的程序应该怎么写呢？其实很简单就是在AndroidManifest.xml清单文件中配置<android-alias>标签。

怎么使用呢？就按上面提到的来写个例子，程序很简单，就是一个MainActivity作为主界面，里面有一个按钮，点击后可以跳转到QRActivity界面(假的“扫一扫”)，并且为QRActivity添加一个在 launcher 中的入口。

首先来看MainActivity.java类，这个里面很简单，其实就是一个按钮，点击后跳转到QRActivity。

public class MainActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
    }

    public void onClick(View view) {
        startActivity(new Intent(this, QRActivity.class));
    }
}

主要看AndroidManifest.xml文件

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
          package="com.yuan.myapplication">

    <application
        android:allowBackup="true"
        android:icon="@mipmap/ic_launcher"
        android:label="主入口"
        android:roundIcon="@mipmap/ic_launcher_round"
        android:supportsRtl="true"
        android:theme="@style/AppTheme">
        <activity android:name=".MainActivity">
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>

                <category android:name="android.intent.category.DEFAULT"/>
                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity>
        <activity android:name=".QRActivity"/>

        <activity-alias
            android:name="QRActivity"
            android:icon="@mipmap/ic_launcher_round"
            android:label="扫一扫"
            android:targetActivity=".QRActivity"
            >
            <intent-filter>
                <action android:name="android.intent.action.MAIN"/>

                <category android:name="android.intent.category.LAUNCHER"/>
            </intent-filter>
        </activity-alias>
    </application>

</manifest>

看一下运行结果：

简单看一下清单文件，可以看到activity-alias标签里面可以像 Activity 一样定义一些常用的属性，比如name、label、icon/等等，其实就可以把它当成一个 Activity 的声明，但有一点要注意，android:targetActivity所对应的 Activity 必须得在AndroidManifest.xml文件中声明，比如上面例子中对 QRActivity 的声明<activity android:name=".QRActivity"/>，也就是说activity-alias并不具有声明 Activity 的能力。需要注意的是activity-alias的必须添加一个这样的intent-filter：

 <intent-filter>
    <action android:name="android.intent.action.MAIN"/>
    <category android:name="android.intent.category.LAUNCHER"/>
 </intent-filter>

否则不能成功。 还有一定需要注意的在主入口的intent-filter或者在activity-alias的intent-filter中添加<category android:name="android.intent.category.DEFAULT"/>，用来告诉程序默认的启动 Activity是哪个，比如其它程序通过包名调用的话，知道默认启动的 activity 是哪个，一般默认会加在主入口的 activity 中。

写完简单用法，再看一下开发文档中对activity-alias常用的属性配置：

<activity-alias android:enabled=["true" | "false"]
                android:exported=["true" | "false"]
                android:icon="drawable resource"
                android:label="string resource"
                android:name="string"
                android:permission="string"
                android:targetActivity="string" >
    . . .
</activity-alias>

one by one 的看一下：

  • android:enabled 这个属性的意思是是否让的 targetActivity 来设置启动，直译的好拗口，直接意思就是就是的这个属性设置 false，它就不能用了。默认值是true，当然 Application 也有这个设置默认也是true,当 Application 和 Activity 的这个属性都为 true 的时候，activity-alias才能起作用。
  • android:exported 其它 APP 能否通过这个alias来启动 targetActivity。如果设置为 false，其它APP 的组件不能启动，但本 App 的的组件却还是可以启动的。
  • android:icon 顾名思义就是可以设置一个独立的 icon，用于显示在 launcher 中。
  • android:label 可以设置一个独立的 label 。
  • android:name 这个跟<activity>标签的name 不一样，暂没有发现其太大意义，可以随便起名，它也不会被其它类引用。
  • android:permission Target Activity的权限设置。
  • android:targetActivity 前面已经说过了，启动的目标 activity。

至此，基本对<activity-alias>标签的使用就搞明白了。
